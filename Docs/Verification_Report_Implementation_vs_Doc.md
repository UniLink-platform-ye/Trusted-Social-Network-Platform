# تقرير التحقق: التوثيق مقابل التنفيذ وقاعدة البيانات

مراجعة بناءً على **UniLink_documentions.md** وملفات **API** و**sql**.

---

## 1. تطابق الـ API مع قاعدة البيانات (تم إصلاحه)

### مشاكل كانت موجودة وتم تصحيحها

| الملف | المشكلة | التصحيح |
|-------|---------|---------|
| **posts** | الـ API كان يستخدم عمود `post_type` بينما الجدول فيه `type` فقط | تم تغيير الـ INSERT لاستخدام العمود `type` |
| **posts** | الـ API يستخدم `p.status` و`UPDATE posts SET status='deleted'` والجدول لم يكن يحتوي عمود `status` | تم إضافة عمود `status` عبر ملف **sql/migration_api_schema_fix.sql** (يجب تشغيله مرة واحدة) |
| **groups** | الـ API يفلتر بـ `g.status='active'` والجدول لم يكن يحتوي عمود `status` | تم إضافة عمود `status` في نفس ملف الـ migration |
| **group_members** | الـ API كان يدرج في عمود `role` بينما اسم العمود في الجدول `member_role` | تم تغيير الـ INSERT لاستخدام `member_role` |

### مطلوب منك

تشغيل الـ migration مرة واحدة على قاعدة البيانات:

```bash
mysql -u admin -p trusted_social_network_platform < sql/migration_api_schema_fix.sql
```

أو تنفيذ محتوى الملف من phpMyAdmin. إذا ظهر خطأ "Duplicate column name" لعمود موجود مسبقاً يمكن تجاهله.

## 2. الميزات حسب التوثيق وحالة تنفيذها

### 2.1 إدارة المستخدمين والمصادقة (الفصل 1.6 و 3.2.1)

| الميزة | التوثيق | API / تطبيق | قاعدة البيانات |
|--------|---------|-------------|-----------------|
| تسجيل الدخول + OTP | مطلوب | ✅ login, verify_otp, resend_otp | ✅ users (otp_code, otp_expires_at) |
| إنشاء الحساب (تسجيل) | مطلوب | ✅ register | ✅ users |
| إدارة الملف الشخصي (عرض) | مطلوب | ✅ بيانات من الجلسة/التوكن | ✅ users |
| تحديث الملف الشخصي (تعديل بيانات، صورة، كلمة مرور) | مطلوب | ❌ لا يوجد profile.php | ✅ users (avatar_url وغيرها) |
| صلاحيات حسب الدور (طالب، أستاذ، مشرف، مدير) | مطلوب | ✅ JWT + role في الـ API | ✅ users.role |

**خلاصة:** المصادقة والتسجيل وOTP مكتملة ومتوافقة. تحديث الملف الشخصي من التطبيق (profile API) غير منفذ.

### 2.3 إدارة المحتوى والمنشورات (الفصل 1.6 و 7.3–7.4)

## 4. ما تم تنفيذه في هذه المراجعة

1. **تصحيح API ليتوافق مع schema قاعدة البيانات:**
   - **posts.php:** استخدام عمود `type` بدلاً من `post_type` في INSERT.
   - **groups.php:** استخدام عمود `member_role` بدلاً من `role` في group_members.

2. **إضافة migration لقاعدة البيانات:**
   - **sql/migration_api_schema_fix.sql:** إضافة الأعمدة `posts.status` و`groups.status` حتى تعمل feed و posts و groups كما ي expect الـ API.

3. **هذا التقرير:** توثيق حالة الميزات بالنسبة للتوثيق والـ API وقاعدة البيانات.

---

## 5. توصيات لاحقة (اختيارية)

- تشغيل **sql/migration_api_schema_fix.sql** على أي بيئة تستخدم فيها الـ API.
- إضافة **profile.php** (تحديث بيانات المستخدم، صورة، كلمة مرور) إذا رغبت بدعم كامل لـ "إدارة الملف الشخصي" من التطبيق.
- إضافة **files.php** (رفع/تحميل) لربط مركز الملفات بالتطبيق.
- إضافة **report** من التطبيق (إنشاء بلاغ) إذا رغبت بتمكين المستخدمين من الإبلاغ من الموبايل.
- إضافة جدول تعليقات (مثلاً `comments`) وواجهة API للتعليقات والإعجاب إذا رغبت بتغطية كاملة لمتطلبات التفاعل في التوثيق.
